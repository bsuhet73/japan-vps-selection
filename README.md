## 日本机房的线路江湖：IIJ、BGP、你该认识谁？

选日本VPS，机房位置只是第一步，背后走的什么线路，才是决定你实际体验的关键变量。

简单科普一下日本到中国大陆的几条主流回国线路：

- **IIJ（Internet Initiative Japan）**：日本本土老牌ISP，联通用户走IIJ回程体验相当不错，延迟稳定。电信用户差一些，晚高峰偶尔有波动。整体来说性价比高，是很多日本VPS商家的首选。
- **BGP优化（Border Gateway Protocol）**：通过BGP多线路智能调度，自动选最优路径回国。ZgoVPS的东京机房用的就是这个，标称"China Optimised"，意味着它针对中国方向做了路由优化。
- **软银（SoftBank/BBTEC）**：联通用户的"亲爹线路"，延迟低、速度快，晚高峰也能稳住。但对电信和移动用户就没那么神了。
- **NTT**：日本最大的电信运营商线路，体量大但对中国方向不算最优，晚高峰容易拥堵，属于"白天法拉利、晚上拖拉机"的类型。

ZgoVPS目前在日本的布局很有意思——**东京走BGP中国优化线路，大阪走IIJ线路**。两种策略，打法完全不同：

- 东京BGP：适合**需要低延迟访问中国大陆**的场景，100Mbps虽然不"大"，但胜在"稳"和"快"。
- 大阪IIJ：走的是"**带宽换线路**"路线——400Mbps起步、最高800Mbps，管够；但IIJ本身不针对中国做特别优化，所以你要的是"流量大"而非"延迟极低"的话，这里就很对味。

> 一句话总结：**要快选东京，要猛选大阪。**
